HTML

  • The head element contains information about the webpage
  • The body element represents information shown to the user
  • There are headers from h1 to h6 with each number indicating how its portrayed to the user with h1 getting the most attention and h6 the least
  • The img element contains src which is the location of the image and alt which is a description of the image
  • All list items have li but before li is ul for bulleted/unordered items or ol for numbered/ordered items
  • meta contains information about the webpage but is not visible to the user
  • The title element shows the name on the webpage tab
  • There can be multiple h1, h2, etc. elements

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • * applies to all elements
  • Link element located in the Head element of HTML links HTML and external CSS together
  • CSS card attribute links class element in Section element of HTML
  • touch filename.css in command line creates CSS file

Git

  • git status: indicates what branch we are currently on
  • git checkout: used to switch branches
  • git checkout -b branch name: used to create a new branch
  • git pull origin branch name: used to pull the newest version of the branch
  • git commit -m "message indicating what changes were made": used to commit any changes made
  • git add: adds current modifications in current branch to staging area
  • git add -A: adds all changes in current branch to staging area
  • git push origin branch name: pushes changes to remote branch
  • git flow is executed in this order: git add -A, git commit -m "message here", git pull origin main, git push origin branch name, and then the request is pulled on GitHub

JavaScript

  • Script element in Footer element links Javascript document to HTML
  • touch filename.js in command line creates Javascript
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • console.log is used to ouput a message to the console by adding an argument which is in parentheses after console.log
  • lines in JavaScript end in semicolons
  • If statements are applied to JavaScript if the condition is truthy
  • Else if statements are applied to JavaScript if any of the conditions not following initial if statement is truthy
  • === checks to see if two values are equal
  • An array is created same as a single data variable but the items in the group are grouped in brackets
  • The first index in an array is always going to be 0
  • A for loop is used to perform a task on all items in an array
  • A for loop has three statements: a starting point which is usually x=0, a condition that applies the array's length which is usually the length property, and the final statement that iterates over each item which is x++
  • /* allows code to be commented out so it's still in the code but not visible
  • A function only works if called by its name to execute it
  • A function is set up in the same way as a variable except parentheses always follows a function's name
  • To call a function, the function's name is used where we want the code executed
